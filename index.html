Project JQK User Manual

Version: 1.0 | Last Updated: 2/10/2026

Welcome to the official user guide for Project JQK . This document is designed to help you install, configure, and utilize the system effectively. Whether you are a first-time user or an administrator, this guide covers the essential steps to get you up and running.

Note: This project was created as a Senior Capstone requirement. If you encounter bugs not listed in the FAQ, please contact the developer at nguye2kh@mail.uc.edu .

1. Installation & Setup

Before using Project JQK, ensure your device meets the following requirements:

  • Operating System: Windows 10+, macOS, or Linux
  • Internet Connection: Required for initial setup
  • Browser: Chrome, Firefox, or Edge (latest versions)

Step-by-Step Installation

  1. Download the latest release package from the repository.
  2. Unzip the file to your desired folder.
  3. Open the terminal (or command prompt) and navigate to the folder.
  4. Run the following command to install dependencies:
npm install
npm start

Once the installation is complete, you should see a success message indicating the server is running on localhost:3000.

3. Glossary of Terms

Here are definitions for technical terms used within this system:

  • API (Application Programming Interface): The tool that allows our system to talk to the database.
  • CRUD: An acronym standing for Create, Read, Update, and Delete. These are the four basic actions you can perform on data in this app.
  • Latency: The slight delay that may occur between clicking a button and seeing the result.

4. Frequently Asked Questions (FAQ)

I forgot my password. How do I reset it?

Currently, password reset must be done by an administrator. Please contact support to request a temporary password.

Why is the "Submit" button greyed out?

The submit button is disabled until all required fields (marked with red asterisks) are filled out correctly.

Does this work on mobile phones?

Yes! The layout is responsive, though for complex data entry, we recommend using a tablet or desktop computer.

I see a "404 Error" message. What does that mean?

This means the page you are looking for cannot be found. Try refreshing your browser or returning to the dashboard.
