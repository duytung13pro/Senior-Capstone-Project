Copilot Instructions

Version 2.0 | Rewood Full-Stack Chinese Learning Platform

Project Rewood is a specialized platform with role-based portals (Student, Teacher, Admin, Center) built using Next.js 15 and Spring Boot 3.4.4.

Critical: This documentation is optimized for AI Pair Programming (Copilot). Always reference the middleware.ts when modifying dashboard routes.

Architecture Stack

Frontend Next.js 15

  • Path: /frontend
  • Patterns: App Router, Route Groups (student), Server Actions.
  • Styling: TailwindCSS + Radix UI components.

Backend Spring Boot 3.4.4

  • Path: /backend
  • Port: 8080 (API endpoints under /api/*).
  • Persistence: MongoDB (Port 27018 via Docker).

Development Workflows

Local Start-up

To run the services independently for debugging:

# Terminal 1 (Backend)
cd backend && mvn spring-boot:run

# Terminal 2 (Frontend)
cd frontend && npm run dev

Docker Orchestration

To build the entire stack (DB, API, Web):

docker-compose up --build

Role-Based Access Control (RBAC)

The system enforces security via a hybrid cookie/session approach:

  1. Login: Backend returns role (TEACHER, STUDENT, etc.).
  2. Enforcement: middleware.ts reads userRole cookie and redirects unauthorized role-group access.
  3. Server Actions: Uses requireRole() from /lib/auth/rbac.ts to gate database operations.
